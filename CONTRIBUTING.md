# Contributing Guidelines

Thank you for your interest in contributing to this wireless security lab project!

## Purpose of the Project

This repository is focused on **ethical, authorized Wi-Fi security testing** and
documenting learning around tools like `aircrack-ng` on Kali Linux.

Contributions should:
- Improve documentation or explanations
- Add safer examples or clarifications
- Fix typos, formatting, or structure
- Suggest better defensive practices

## What NOT to Contribute

Please do **NOT** submit:
- Real captured `.cap` or `.pcap` files
- Actual passwords, SSIDs, BSSIDs or MAC addresses
- Instructions that encourage illegal or unauthorized hacking

Any sensitive or illegal content will be removed.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Make your edits.
4. Open a Pull Request with a clear description of:
   - What you changed
   - Why it improves the project
